## 什么是 LingProxy？

LingProxy 是一个高性能的 AI API 网关，专为管理和代理各种 AI 服务提供商的 API 调用而设计。它提供了 OpenAI 兼容接口、智能负载均衡、请求路由和全面的管理功能。

## 核心特性

### 🚀 核心能力
- **OpenAI 兼容 API**：与 OpenAI SDK 和兼容客户端无缝集成
- **流式响应支持**：完整支持 Server-Sent Events (SSE) 流式响应，适用于聊天补全接口
- **智能负载均衡**：多种路由策略（轮询、随机、加权等）
- **自动重试机制**：可配置的自动重试功能，支持指数退避策略
- **请求路由**：基于策略的请求路由，支持模型匹配和故障转移
- **请求日志**：完整的请求链路追踪和日志记录

### 🔐 安全与认证
- **灵活认证机制**：全局认证开关，可按端点配置
- **管理员管理**：用户名/密码认证，安全密码哈希存储
- **API Key 管理**：请求端 API Key 管理，支持策略关联
- **API 密钥认证**：基于 API 密钥的安全认证
- **CORS 支持**：可配置的跨域资源共享

### 📊 管理功能
- **管理后台**：基于 Vue 3 + Element Plus 的现代化 Web 管理界面
- **国际化支持**：前端界面完整支持中英文切换
- **LLM 资源管理**：基于驱动的架构（目前支持 OpenAI 驱动）
- **模型管理**：灵活的模型配置，支持定价和使用限制
- **策略管理**：内置路由策略模板和自定义策略实例，随机选择策略支持LLM资源池配置
- **请求管理**：完整的请求记录和追踪功能
- **用量统计**：按LLM资源分组的详细用量统计，包括Token使用量、请求数、成功率等指标
- **系统设置**：动态配置管理
- **系统监控**：实时系统信息和统计
- **日志管理**：查看和管理系统日志，支持过滤和搜索功能

## 使用场景

- **多提供商 AI 服务**：通过统一接口管理多个 AI 服务提供商
- **负载均衡**：在多个资源间分配请求以提高性能
- **请求路由**：根据策略将请求路由到不同的提供商
- **API 网关**：作为 AI 服务 API 的网关，提供认证和日志功能
- **开发与测试**：用作开发和测试环境的代理

## 架构

LingProxy 采用现代微服务架构：

- **前端**：Vue 3 + Element Plus + Vite
- **后端**：Go + Gin + GORM
- **存储**：SQLite（默认）/ MySQL / PostgreSQL
- **API**：RESTful API + OpenAI 兼容 API
